# HAG-DTA 补充实验全景

本文档用于说明：在原论文基础上，后续补充实验到底补了什么、为什么要补、每组实验在论文逻辑中承担什么作用。它的定位不是运行手册，而是论文结果部分和返修说明的写作骨架。后续只需要把实际数值填入对应位置，就可以继续完善论文内容。

## 1. 补充实验的总体目的

原论文已经提出 HAG-DTA 的核心思想：通过层级图结构建模和多源特征融合提升药物-靶标相互作用预测性能。补充实验不改变模型主张，而是补齐审稿人容易追问的证据链：

1. **性能是否可靠**：主结果需要多随机种子支撑，而不是单次运行结果。
2. **比较是否公平**：回归任务要和图模型 baseline 对齐，分类任务要和序列模型 baseline 对齐。
3. **提升来自哪里**：需要说明优势不是因为换了 backbone、换了划分或偶然调参。
4. **超参数是否敏感**：层级池化节点数 `n1/n2` 和 MMD 系数 `beta` 是否会显著影响结论。
5. **论文如何补写**：把新增结果映射到主结果表、统计检验、消融实验和讨论部分。

补充实验的主线可以概括为：

> 先建立公平 baseline，再用多 seed 和统计检验证明性能稳定，最后用敏感性分析说明 HAG-DTA 的收益来自层级结构和局部-全局约束，而不是偶然的数据划分、backbone 选择或超参数调优。

## 2. 统一实验协议

| 数据集 | 任务 | 划分方式 | 主要指标 | 对齐对象 |
|---|---|---|---|---|
| Davis | 回归 | DeepDTA classic split | MSE / CI / rm2 | GraphDTA / DeepDTA 系列 |
| KIBA | 回归 | DeepDTA classic split | MSE / CI / rm2 | GraphDTA / DeepDTA 系列 |
| Human | 二分类 | 80/10/10 random split，划分 seed=1234 | AUROC / AUPRC / Precision / Recall | Tsubaki / TransformerCPI |
| C.elegans | 二分类 | 80/10/10 random split，划分 seed=1234 | AUROC / AUPRC / Precision / Recall | Tsubaki / TransformerCPI |

主实验统一采用 5 个训练随机种子：

```text
100, 1000, 2000, 3000, 4000
```

敏感性分析默认采用单 seed。原因是敏感性分析的目标是观察趋势和稳定区间，不替代主性能表；如果后续某个敏感性结论特别关键，再对代表组合补多 seed 即可。

分类数据统一采用 80/10/10 random split。这样可以与 Tsubaki 和 TransformerCPI 的实验路线对齐，同时避免旧的 fold 或 64/16/20 划分造成论文口径混乱。

## 3. 主实验一：Davis / KIBA 回归任务

### 3.1 实验要回答的问题

这一组实验回答：在 Davis 和 KIBA 两个经典回归数据集上，HAG-DTA 是否优于非层级图神经网络 baseline；如果使用相同 GNN backbone，优势是否仍然存在。

### 3.2 设计逻辑

回归任务中，GraphDTA 是最重要的补充 baseline。它同样使用分子图作为药物输入，并且可以使用 GCN、GAT、GIN、SAGE 等常见图神经网络 backbone。HAG-DTA 也对齐这些 backbone 做比较。

这个设计的核心是控制变量：

| 比较 | 控制变量 | 能说明的问题 |
|---|---|---|
| HAG-DTA GIN vs GraphDTA GIN | 相同 backbone | HAG-DTA 的收益是否来自层级结构，而不是 GIN 本身 |
| HAG-DTA GCN/GAT/SAGE vs GraphDTA GCN/GAT/SAGE | 相同 backbone | 层级建模是否对不同 GNN 算子都有效 |
| Davis 和 KIBA 同时验证 | 不同回归数据集 | 结论是否具有数据集稳定性 |

如果 HAG-DTA 在同一 backbone 下仍然优于 GraphDTA，就可以把论文结论写成：性能提升主要来自层级池化、多尺度分子表示和局部-全局表示约束，而不是来自简单更换 GNN 算子。

2. 这种优势在相同 backbone 下仍然存在，因此不是 backbone 选择带来的偶然优势。
3. 多 seed 结果和显著性检验可以支持模型性能稳定性的论断。

## 4. 主实验二：Human / C.elegans 分类任务

### 4.1 实验要回答的问题

这一组实验回答：在 Human 和 C.elegans 二分类任务上，HAG-DTA 是否优于强序列模型 TransformerCPI；如果双方使用相同原始样本和相同 80/10/10 划分，HAG-DTA 的优势是否仍然成立。

### 4.2 设计逻辑

分类任务中，TransformerCPI 是主要 baseline。它代表了基于序列和 self-attention 的 CPI 建模路线，与 HAG-DTA 的图结构层级建模形成互补对比。

公平性重点有两个：

| 问题 | 处理方式 | 论文意义 |
|---|---|---|
| 样本集合是否一致 | HAG-DTA 和 TransformerCPI 使用同一批原始样本 | 避免结果差异来自预处理过滤 |
| 数据划分是否一致 | 使用同一个 80/10/10 random split | 避免训练/测试样本不同造成偏差 |

TransformerCPI 官方 Human/C.elegans 设置是 40 个训练 epoch。虽然 epoch 数不多，但该模型训练过程逐样本处理，并且每个 epoch 都进行验证和测试，因此实际运行时间可能较长。论文中只需要说明它使用了官方训练口径，不需要展开工程实现细节。

## 5. 补充实验一：n1 / n2 层级池化敏感性

### 5.1 实验要回答的问题

HAG-DTA 的层级池化需要指定两个结构超参数：

| 参数 | 含义 |
|---|---|
| `n1` | 第一层池化后的超节点数，近似对应局部官能团/片段粒度 |
| `n2` | 第二层池化后的超节点数，近似对应更粗粒度药效团或分子区域 |

审稿人可能会问：模型性能是否依赖某个精细调出来的 `n1/n2` 组合。如果只有一个组合表现好，说明模型鲁棒性不足；如果一片区域表现稳定，说明层级池化设计更可信。

### 5.2 设计逻辑

四个数据集采用相同网格：

```text
n1 ∈ {4, 5, 6, 7, 8}
n2 ∈ {2, 3, 4}
约束 n2 < n1
共 14 组
```

主实验默认保持：

```text
n1 = 6
n2 = 2
```

敏感性分析使用单 seed。它的论文作用不是证明某个组合是全局最优，而是证明默认组合附近存在稳定性能区域，从而降低“超参数过拟合”的质疑。

## 6. 补充实验二：MMD loss 系数 beta

### 6.1 实验要回答的问题

HAG-DTA 使用 MMD loss 约束局部结构表示和全局结构表示之间的分布差异。补充 beta 实验要回答：

1. 去掉 MMD 后性能是否下降。
2. 默认 beta 是否只是偶然调参结果。
3. beta 增大或减小时，模型是否仍然稳定。

### 6.2 设计逻辑

beta 搜索建议在 Davis 上进行，使用 HAG-DTA GIN：

```text
beta ∈ {0, 0.01, 0.05, 0.1, 0.5, 1.0}
```

其中 `beta=0` 可以视为去除 MMD loss 的组件消融；非零 beta 用于观察局部-全局分布对齐强度变化时模型是否稳定。

### 6.3 论文中建议新增的表格

**表 G：MMD beta 敏感性分析**

| beta | MSE ↓ | CI ↑ | rm2 ↑ | Interpretation |
|---:|---:|---:|---:|---|
| 0 | 待补 | 待补 | 待补 | 去除 MMD |
| 0.01 | 待补 | 待补 | 待补 | 弱正则 |
| 0.05 | 待补 | 待补 | 待补 | 默认设置 |
| 0.1 | 待补 | 待补 | 待补 | 中等正则 |
| 0.5 | 待补 | 待补 | 待补 | 强正则 |
| 1.0 | 待补 | 待补 | 待补 | 更强正则 |

### 6.4 论文中可写的结论方向

如果 `beta=0` 明显弱于非零 beta，可以说明 MMD 对齐确实有贡献；如果多个非零 beta 表现接近，可以说明 HAG-DTA 不依赖极窄的 beta 取值。
