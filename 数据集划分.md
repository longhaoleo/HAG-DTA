## 1. Tsubaki 原始 GNN-CNN 论文怎么划分

Tsubaki 的 Bioinformatics 论文使用 Human 和 C.elegans 两个 CPI 数据集，数据来自 Liu et al. 构建的高可信负样本数据集。论文中说明 Human 包含 3369 个正相互作用，C.elegans 包含 4000 个正相互作用。

先固定随机种子 `1234` 打乱数据，然后：

```python
dataset_train, dataset_ = split_dataset(dataset, 0.8)
dataset_dev, dataset_test = split_dataset(dataset_, 0.5)
```

也就是：

| 部分             | 比例 |
| ---------------- | ---- |
| Train            | 80%  |
| Validation / Dev | 10%  |
| Test             | 10%  |

所以，**Tsubaki 原始实现是 random split 80/10/10**，并且使用 dev set 监控 AUC，test set 报告 AUC、Precision、Recall。([GitHub](https://raw.githubusercontent.com/masashitsubaki/CPI_prediction/master/code/run_training.py))

## 2. TransformerCPI 怎么划分

TransformerCPI 使用 Human、C.elegans 和 BindingDB 三个 benchmark 数据集。它在数据集描述中明确写到，Human 和 C.elegans 的 **training、valid 和 test sets are randomly split**，并且引用 Tsubaki et al. 的设置。

所以 TransformerCPI 的分类任务逻辑可以理解为：

| 数据集    | 划分方式                                       |
| --------- | ---------------------------------------------- |
| Human     | 随机 train / valid / test split                |
| C.elegans | 随机 train / valid / test split                |
| BindingDB | 使用预设 split，测试未知 ligand / protein 泛化 |

它和 Tsubaki 是同一路线，适合写作时表述为：

> Following Tsubaki et al. and TransformerCPI, the Human and C.elegans datasets were randomly split into training, validation, and test sets.

如果你想写具体比例，建议基于 Tsubaki 官方代码写 **80/10/10**。

## 3. MGraphDTA 怎么划分

MGraphDTA 的 Human/C.elegans 分类实验主要对齐的是 GNN-CNN、GraphDTA、TrimNet、VQA-seq 和 TransformerCPI。它文中说明：为了公平比较，分类任务使用了与前人研究相同的数据集和数据划分方案，并且所有实验重复 3 次，每次使用不同随机种子，最终报告 mean 和 standard deviation。([ResearchGate](https://www.researchgate.net/publication/357616909_MGraphDTA_deep_multiscale_graph_neural_network_for_explainable_drug-target_binding_affinity_prediction?utm_source=chatgpt.com))

MGraphDTA 表 2 的 Human/C.elegans 分类结果包括：

| Dataset   | Model          | Precision     | Recall        | AUC           |
| --------- | -------------- | ------------- | ------------- | ------------- |
| Human     | GraphDTA       | 0.882 (0.040) | 0.912 (0.040) | 0.960 (0.005) |
| Human     | TransformerCPI | 0.916 (0.006) | 0.925 (0.006) | 0.973 (0.002) |
| Human     | MGNN-MCNN      | 0.955 (0.005) | 0.956 (0.003) | 0.983 (0.003) |
| C.elegans | GraphDTA       | 0.927 (0.015) | 0.912 (0.023) | 0.974 (0.004) |
| C.elegans | TransformerCPI | 0.952 (0.006) | 0.953 (0.005) | 0.988 (0.002) |
| C.elegans | MGNN-MCNN      | 0.980 (0.004) | 0.967 (0.005) | 0.991 (0.001) |

这里更像是沿用前人 random split 方案，然后用 3 个随机种子重复实验，而非标准 5-fold CV。

## 4. 后续论文中也有使用 5-fold 或 10-fold 的

后续一些 DTI 分类论文采用了交叉验证。

例如 ParaCPI 明确写到在 Human 和 C.elegans 上使用 **5-fold cross-validation** 验证模型效果。([ACM Digital Library](https://dl.acm.org/doi/pdf/10.1109/TCBB.2024.3404889?utm_source=chatgpt.com))

MIFAM-DTI 使用 Human 和 C.elegans 两个数据集，说明两者均来自 Liu et al.，正负样本比例保持 1:1，并在实验部分写明对两个数据集使用 **10-fold cross-validation**，评价指标包括 AUC、AUPR、Precision、Recall 和 F1-score。([Frontiers](https://www.frontiersin.org/journals/genetics/articles/10.3389/fgene.2024.1381997/full))

所以你采用 5-fold CV 并不突兀，只是要说明它对齐的是后续分类实验常用的稳定评估方式，而不是 Tsubaki 原始代码的 80/10/10 单次 random split。

## 5. AMMVF-DTI 的情况

AMMVF-DTI 使用 Human、C.elegans 和 DrugBank，并说明 Human 和 C.elegans 数据来自 Liu et al.，正样本来自 DrugBank 和 Matador，负样本通过 dissimilar rules 筛选，正负比例约为 1:1。([MDPI](https://www.mdpi.com/1422-0067/24/18/14142))

它的结果表报告了 AUC、Precision、Recall，并且对部分模型给出了 mean ± std，例如 GraphDTA、GCN、TransformerCPI 等。([MDPI](https://www.mdpi.com/1422-0067/24/18/14142))

但从正文可见内容看，它对 Human/C.elegans 的具体 split 比例没有像 Tsubaki 代码那样明确写成 80/10/10，也没有像 MIFAM-DTI 那样明确写 10-fold。因此如果你引用 AMMVF-DTI 的结果，建议只说“reported results”，不要强行说它的划分和你完全一致。
